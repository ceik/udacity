APIs

Application Programming Interfaces or APIs specify a way in which a programm can interact with a website or webservice in a more machine-friendly way (than reading an HTML file). While APIs can come in many form, SOAP and REST are two very popular standards. In both protocols HTTP methods are used for communication. Data is often returned in JSON or XML format. E.g. Twitter, Github, or Asana provide REST APIs.

HTTP responses are being sent back by the server as a reply to an HTTP request. Each response consists of a header and a body. While the body contains the actual content of the response, e.g. the HTML or JSON document that was requested, the header contains various meta data that can be very useful. The most important part of the header is the status code and the reason phrase that goes with it. Notable examples are 200 - OK or 404 - Not Found. In general any code starting with 2 indicates success, while 4 indicates client errors (e.g. 404: Asking for something that doesn't exist), and 5 indicates server errors.

Recursion

In computer science recursion refers to a method where the solution to a problem depends on the solutions to smaller instances of the same problem. This stands opposed to iteration, which means the repition of a block of code (e.g. via a for- or while-loop). Some functional languages, like Haskell, do not have loops and rely entirely on recursion to solve iterative problems.

In Python recursion usually looks like "calling a function within itself". A simple example problem for this is computing the factorial. Recursive functions have at least one base case , for which the function simply computes and outcome. In the case of the factorial this would be 0! = 1. For all other cases, the recursive cases, the function calls itself. E.g. n! = n(n-1)! for n > 0 for our factorial example.

In order to prevent an infinite loop it is imperative that either the base case is reached at some point, or that a stopping condition is included in the function. An example use case for the latter would be a function that calculates the number pi.

Another interesting application of recursion are recursive data structures. These are structures that can can grow dynamically during the execution of a program. For example linked lists or tree structures fit this definition.

Lesson 1.1 - The Basics of the Web and HTML

The WWW is mainly made up out of HTML (Hypertext Markup Language) documents, that are linked together via Hyperlinks. Browsers sends requests to servers via the internet and the HTTP protocol. Servers answer with documents, often in HTML.

HTML Components are mainly Text Content and Markup.

Markup elements usually have an opening and closing tag. The part between the tags is called content. If and element consists of only one tag, e.g. the image tag, it is called a void element. Markup elements can be either inline (e.g. <span>) or block (e.g. <div>). Block tags create a box around the content.

For an extensive HTML reference see here.

Lesson 1.2 - Creating a Structured Document

Elements that are not void are defined by an opening and closing tag. The content and can be empty, text, or another element. Tags can have attributes with values. Examples of attributes are class or href, e.g.: <a class="link" href="http://www.google.com">

HTML tags are being turned into elements by the browser. Elements can have other elements inside them, which leads to a tree-like structure of boxes (because all elements are boxes). This structure is defined by the Document Object Model or DOM. The DOM is the standard convention for representing and interacting with elements in HTML.

When thinking about the layout of a page it is essential to think in terms of boxes, because every element on the page will be a box.

Lession 1.3 - Adding CSS Style and HTML Structure

CSS (Cascading Style Sheets) are an important addition to HTML. They provide an efficient way of changing the style of the page.

In order to use CSS a few things should be kept in mind. The head of the html document needs to contain a link to the .css file in this format: <link href="notes_stage_1.css" rel="stylesheet">. Also it is very handy to give html elements a class attribute and give the same class to similar elements.

For an extensive CSS reference see here.

HTML does not have escape characters. Instead ASCII code needs to be used for certain characters, e.g. <, &, and ". ASCII codes have the following structure: &#num; where num stands for a number or an alternative abbreviation. E.g. the num/abbreviation for the characters above are 60/lt, 38/amp, and 34/quot respectively.

Avoiding repetition is important for a variety of reasons. Avoiding Errors: When programmers don't avoid repetition, they will often have to do the same thing over and over. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere the repeated code appears, problems will arise. Consistency: In this web page, I wanted every lesson to look the same. Instead of manually giving each one the same style, I was able to assign each lesson div the same class and then specify the style for that class in only one place! By avoiding repetion, it's now very easy for me to make changes to this style.

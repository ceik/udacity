Servers

Nearly the whole Internet is made up of servers that work according to the client-server model. Servers are programs that often run on dedicated machines. There are many different types of servers: Application servers, mail servers, or web servers. The latter wait for and respond to client requests that come via the HTTP protocol, like GET or POST.

Input Validation

Data that is submitted by users via forms imbedded into the site needs to be validated. For example html code could be submitted via the form, which might break the site (or worse). One way of validating is to escape all characters that could interact with the code. Most webapp frameworks have a global setting to do this.

Templates

Webapplications can and should use HTML templates in order to avoid repitition. Together with control flow structures they can be used by the application to dynamically construct the HTML code for the site that the user sees. Google App Engine uses the jinja2 library , which lets you create pages from a template file and a dictionary.

Databases & Google App Engine Datastore

ACID:
  • Atomicity: All parts of a transaction succeed or fail together
  • Consistency: The database will always be consistent
  • Isolation: No transaction can interfere with anothers
  • Durability: Once the transaction is comitted, it won't be lost
Google App Engine Datastore:

The datastore IDs are made up of strings or numbers. It contains entities whose columns are not fixed. Every entity has an id and is defined by parent-ancestor relationships .
